Detailed Description

This namespace provides implementations of various memory reclamation schemes.

The following schemes are implemented:

  • lock_free_ref_count
  • epoch_based
  • new_epoch_based
  • quiescent_state_based
  • debra
  • hazard_pointer
  • hazard_era
  • stamp_it

enum xenium::reclamation::region_extension
strong

Defines whether the size of a critical region can be extended using region_guards.

Enumerator
none 

Critical regions are never extended, i.e., region_guards are effectively no-ops.

eager 

The critical region is entered upon construction of the region_guard and left once the region_guard gets destroyed.

lazy 

A critical region is only entered when a guard_ptr is created. But if this is done inside the scope of a region_guard, the critical region is only left once the region_guard gets destroyed.
